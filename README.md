# PaddleOCR VL Fine-tuning POC

Fine-tune a PaddleOCR recognition model (`SVTR_LCNet` / PP-OCRv4 backbone) on
domain-specific scanned tabular documents — specifically HGF-format 1500-row
ledger sheets.

The pipeline goes from a raw GIF scan + Excel ground-truth all the way to a
trained, exported inference model in one command.

---

## 5 · Running the fine-tuner

### Quick start (uses built-in sample data)

```bash
python train.py
```

This runs the full pipeline:
1. Extracts cell crops from `sample input/HGF-1500-1887.gif`
2. Matches crops to ground-truth labels from `original data/3.HGF-1500-1887_Original_Data.xlsx`
3. Filters blank-cell crops
4. Fine-tunes the SVTR_LCNet recognition model
5. Exports the best checkpoint to inference format

### Custom data sources

```bash
python train.py \
  --images "path/to/scan1.gif,path/to/scan2.gif" \
  --ground-truths "path/to/gt1.xlsx,path/to/gt2.xlsx"
```

### Override hyper-parameters

```bash
python train.py \
  --epochs 80 \
  --batch-size 16 \
  --lr 1e-4 \
  --warmup-epochs 5 \
  --output-dir runs/experiment_v2
```

### Resume from a checkpoint

```bash
python train.py --pretrained-model runs/finetune_v1/models/best_accuracy
```

### Skip export / evaluation after training

```bash
python train.py --no-export --no-eval
```

## 6 · Full CLI reference

```
usage: train.py [-h]
                [--images IMAGES] [--ground-truths GROUND_TRUTHS]
                [--pipeline-output PIPELINE_OUTPUT]
                [--val-ratio VAL_RATIO]
                [--output-dir OUTPUT_DIR]
                [--epochs EPOCHS] [--batch-size BATCH_SIZE]
                [--lr LR] [--warmup-epochs WARMUP_EPOCHS]
                [--weight-decay WEIGHT_DECAY]
                [--img-height IMG_HEIGHT] [--img-width IMG_WIDTH]
                [--max-text-length MAX_TEXT_LENGTH]
                [--num-workers NUM_WORKERS]
                [--pretrained-model PRETRAINED_MODEL]
                [--no-gpu]
                [--no-grid-slicing] [--expected-cols EXPECTED_COLS]
                [--crop-upscale CROP_UPSCALE]
                [--keep-empty-labels]
                [--no-export] [--no-eval]
```

| Flag | Default | Description |
|---|---|---|
| `--images` | built-in | Comma-separated GIF/PNG/JPG source paths |
| `--ground-truths` | built-in | Comma-separated Excel ground-truth paths (must match `--images`) |
| `--pipeline-output` | `finetune_data` | Directory for crops, label files, stats |
| `--val-ratio` | `0.15` | Fraction of data reserved for validation |
| `--output-dir` | `runs/finetune_v1` | Root directory for training artefacts |
| `--epochs` | `50` | Training epochs |
| `--batch-size` | `8` | Batch size per GPU/CPU card |
| `--lr` | `5e-5` | Peak learning rate (Cosine schedule) |
| `--warmup-epochs` | `5` | Warm-up epochs before peak LR |
| `--weight-decay` | `1e-5` | L2 regularisation factor |
| `--img-height` | `32` | Recognition model input height (px) |
| `--img-width` | `320` | Recognition model input width (px) |
| `--max-text-length` | `50` | Maximum characters per crop |
| `--num-workers` | `0` | DataLoader workers (keep 0 on Windows) |
| `--pretrained-model` | `None` | PaddleOCR checkpoint directory to resume from |
| `--no-gpu` | off | Force CPU training |
| `--no-grid-slicing` | off | Use PaddleOCR detection instead of grid slicing |
| `--expected-cols` | `25` | Number of table columns (SLNO + 24 data cols) |
| `--crop-upscale` | `2.0` | Bicubic upscale factor for saved cell crops |
| `--keep-empty-labels` | off | Keep blank-cell crops in training (not recommended) |
| `--no-export` | off | Skip exporting the best checkpoint |
| `--no-eval` | off | Skip validation evaluation after training |

## 8 · Accuracy notes

The following design choices were made to maximise recognition accuracy on
dense scanned tabular data:

- **`character_type: en_sensitive`** — preserves uppercase/lowercase, which
  matters for names (`Jos`), IDs (`HGF34401`), and email addresses.
- **`max_text_length: 50`** — covers the observed maximum label length of 45
  characters (addresses) with a safety margin; the old default of 40 silently
  truncated labels.
- **`RecConAug` augmentation** — contrastive augmentation inserted after
  `RecAug` improves robustness on low-resolution scans.
- **Gradient clipping (`ClipGradByGlobalNorm: 5.0`)** — stabilises training
  with small batches.
- **`learning_rate: 5e-5`** — conservative default for fine-tuning; prevents
  overwriting pretrained low-level features.
- **`crop_upscale: 2.0`** — bicubic 2× upscaling of raw cell crops (typical
  row height ~20 px) before saving, producing sharper training images.
- **Empty-label filtering** — blank table cells are excluded from training by
  default; they waste capacity without contributing text-recognition signal.
