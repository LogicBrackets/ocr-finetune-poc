# PaddleOCR Recognition Fine-tuning

Fine-tune a PaddleOCR `SVTR_LCNet` (PP-OCRv4) recognition model on
domain-specific scanned tabular documents — specifically HGF-format
1500-row ledger sheets.

The pipeline takes a raw GIF scan + Excel ground-truth all the way to a
trained, exported inference model in a single command.

## Quick Start

### Option A — Automated setup (recommended)

```bash
# CPU training
bash setup.sh cpu

# GPU training (CUDA 11.7)
bash setup.sh cu117

# GPU training (CUDA 11.8)
bash setup.sh cu118
```

The script creates a `.venv`, installs PaddlePaddle and all dependencies,
then optionally downloads the PP-OCRv4 pretrained weights.

After setup, activate the environment and train:

```bash
source .venv/bin/activate
make download-pretrained   # if not downloaded during setup
make train
```

### Option B — Manual step-by-step

#### Step 1 — Create a virtual environment

**Linux / macOS**
```bash
python3.10 -m venv .venv
source .venv/bin/activate
```

**Windows (PowerShell)**
```powershell
python -m venv .venv
.venv\Scripts\Activate.ps1
# If blocked by execution policy, run once:
# Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
```

#### Step 2 — Install PaddlePaddle

> Install PaddlePaddle **before** `requirements.txt` so pip resolves
> PaddleOCR's transitive dependency correctly.

**CPU only**
```bash
pip install paddlepaddle==2.6.1 -i https://pypi.tuna.tsinghua.edu.cn/simple
```

**GPU — CUDA 11.7**
```bash
pip install paddlepaddle-gpu==2.6.1.post117 \
    -f https://www.paddlepaddle.org.cn/whl/linux/mkl/avx/stable.html
```

**GPU — CUDA 11.8**
```bash
pip install paddlepaddle-gpu==2.6.1.post118 \
    -f https://www.paddlepaddle.org.cn/whl/linux/mkl/avx/stable.html
```

Verify:
```bash
python -c "import paddle; paddle.utils.run_check()"
```

#### Step 3 — Install project dependencies

```bash
pip install -r requirements.txt
```

> **Windows:** If `opencv-python` fails to build, use:
> ```bash
> pip install opencv-python-headless
> ```

Verify:
```bash
python -c "from paddleocr import PaddleOCR; import cv2, pandas; print('All OK')"
```

#### Step 4 — Download PP-OCRv4 pretrained weights

Fine-tuning from a pretrained checkpoint achieves much higher accuracy than
training from scratch and converges significantly faster.

```bash
# Download English mobile model (~11 MB, recommended)
python scripts/download_pretrained.py

# Download English server model (~90 MB, higher accuracy)
python scripts/download_pretrained.py --model server

# List all available models
python scripts/download_pretrained.py --list
```

Weights are saved to `pretrained/en_PP-OCRv4_rec_train/best_accuracy.pdparams`.

#### Step 5 — Run fine-tuning

```bash
python train.py \
    --pretrained-model pretrained/en_PP-OCRv4_rec_train/best_accuracy
```

This runs the full pipeline:
1. Extracts 37,500 cell crops from `sample input/HGF-1500-1887.gif`
2. Matches each crop to its ground-truth label from `original data/3.HGF-1500-1887_Original_Data.xlsx`
3. Filters blank-cell crops (reduces to ~37,376 samples)
4. Copies images and writes label files into `runs/finetune_v1/data/`
5. Generates `runs/finetune_v1/rec_finetune.yml` with absolute paths
6. Launches `tools/train.py` to fine-tune SVTR_LCNet via CTC loss
7. Exports the best checkpoint to `runs/finetune_v1/exported/`
8. Evaluates on the validation split

## CLI Reference

```bash
python train.py [OPTIONS]
```

### Data sources

| Flag | Default | Description |
|---|---|---|
| `--images` | built-in | Comma-separated GIF/PNG/JPG source scan paths |
| `--ground-truths` | built-in | Comma-separated Excel paths (must match `--images` order) |
| `--pipeline-output` | `finetune_data` | Directory for crops, label files, stats |
| `--val-ratio` | `0.15` | Fraction of data reserved for validation |

### Training hyper-parameters

| Flag | Default | Description |
|---|---|---|
| `--output-dir` | `runs/finetune_v1` | Root directory for all training artefacts |
| `--epochs` | `50` | Training epochs |
| `--batch-size` | `8` | Batch size per GPU/CPU card |
| `--lr` | `5e-5` | Peak learning rate (Cosine schedule with warmup) |
| `--warmup-epochs` | `5` | Linear warm-up epochs before peak LR |
| `--weight-decay` | `1e-5` | L2 regularisation factor |
| `--img-height` | `32` | Recognition model input height (px) |
| `--img-width` | `320` | Recognition model input width (px) |
| `--max-text-length` | `50` | Maximum characters per crop (must be ≥ max label length) |
| `--num-workers` | `0` | DataLoader workers (keep `0` on Windows) |
| `--pretrained-model` | `None` | Path to a PaddleOCR checkpoint base (e.g. `pretrained/.../best_accuracy`) |
| `--no-gpu` | off | Force CPU even when CUDA is available |

### Data pipeline

| Flag | Default | Description |
|---|---|---|
| `--no-grid-slicing` | off | Fall back to PaddleOCR detection + row-clustering instead of grid slicing |
| `--expected-cols` | `25` | Number of table columns (SLNO + 24 data columns) |
| `--crop-upscale` | `2.0` | Bicubic upscale factor applied to each saved cell crop |
| `--keep-empty-labels` | off | Keep blank-cell crops in training set (not recommended) |

### Post-training

| Flag | Default | Description |
|---|---|---|
| `--no-export` | off | Skip exporting the best checkpoint to inference format |
| `--no-eval` | off | Skip running evaluation after training |

## Common Recipes

### Fine-tune from pretrained (standard)

```bash
python train.py \
    --pretrained-model pretrained/en_PP-OCRv4_rec_train/best_accuracy \
    --epochs 50 \
    --lr 5e-5
```

### Train with custom scans and ground truth

```bash
python train.py \
    --images "data/scan_A.gif,data/scan_B.gif" \
    --ground-truths "data/gt_A.xlsx,data/gt_B.xlsx" \
    --pretrained-model pretrained/en_PP-OCRv4_rec_train/best_accuracy \
    --output-dir runs/custom_v1
```

### Resume training from a checkpoint

```bash
# Resume from last saved epoch
python train.py \
    --pretrained-model runs/finetune_v1/models/latest \
    --output-dir runs/finetune_v1 \
    --epochs 50

# Resume from a specific epoch
python train.py \
    --pretrained-model runs/finetune_v1/models/iter_epoch_20 \
    --output-dir runs/finetune_v1
```

### Higher batch size for faster training (GPU)

```bash
python train.py \
    --pretrained-model pretrained/en_PP-OCRv4_rec_train/best_accuracy \
    --batch-size 32 \
    --lr 1e-4 \
    --warmup-epochs 5
```

### Dataset build only (inspect crops before training)

```bash
python train.py --no-export --no-eval
```

### Run inference with trained model

```bash
python tools/infer_rec.py \
    -c runs/finetune_v1/rec_finetune.yml \
    -o Global.pretrained_model=runs/finetune_v1/exported \
       Global.infer_img=path/to/cell.png
```

## Architecture

| Component | Choice | Notes |
|---|---|---|
| **Algorithm** | SVTR_LCNet | PP-OCRv4 lightweight recognition pipeline |
| **Backbone** | MobileNetV1Enhance (scale=0.5) | ~11 MB, fast on CPU |
| **Neck** | SequenceEncoder (SVTR, dims=64, depth=2) | Self-attention for sequence context |
| **Head** | CTCHead | CTC loss — no explicit alignment needed |
| **Loss** | CTCLoss | Standard for variable-length text recognition |
| **Metric** | RecMetric (accuracy) | Exact-match accuracy on validation set |

## Accuracy Design Notes

### Character encoding — `en_sensitive`
Case distinctions are preserved because the dataset contains:
- Patient IDs: `HGF34401`
- Names: `Jos`, `EDGE`, `McDonnell`
- Emails: `ndickman@gmail.com`

Switching to `en` (case-insensitive) or `ch` would degrade accuracy.

### `max_text_length: 50`
The observed maximum label length in the dataset is **45 characters**
(long street addresses). The earlier default of `40` silently truncated
these labels during `CTCLabelEncode`, corrupting the training signal for
the longest samples. The value `50` adds a safety margin.

### Learning rate: `5e-5`
Fine-tuning from a pretrained checkpoint requires a conservative learning
rate. Using `1e-4` (the scratch-training default) risks overwriting the
low-level visual features learned during PP-OCRv4 pretraining. `5e-5`
keeps the backbone stable while letting the head adapt.

### Warmup: 5 epochs
A 5-epoch linear warmup prevents optimizer shock at the start of
fine-tuning, when gradient magnitudes are high before the model has
had a chance to adapt to the new domain.

### Gradient clipping: `ClipGradByGlobalNorm(5.0)`
Small batch sizes (8) on dense scanned data can produce occasional large
gradient spikes. Global-norm clipping at 5.0 eliminates these without
affecting normal gradient flow.

### Crop upscaling: `2.0×`
Raw table rows in the source scans are approximately 20 px tall. Saving
crops at their native resolution produces blurry 20×N training images that
are below the 32 px input height the model expects. Bicubic 2× upscaling
before saving produces sharper crops and noticeably improves validation
accuracy on dense scans.

### Empty-label filtering
Blank table cells are removed from training by default (`--keep-empty-labels`
off). They add no text-recognition signal and can confuse the CTC decoder
into predicting empty sequences for short-text crops.

### Grid-based cell slicing (default)
The preferred extraction mode uses OpenCV morphological projection analysis
to detect row/column boundaries directly from the image, then slices every
cell from its exact grid position. This is:
- **Deterministic**: each (row, col) index maps 1-to-1 to an Excel cell
- **Fast**: no PaddleOCR detection pass needed
- **Robust**: handles tables without drawn grid lines

The OCR-detection fallback (`--no-grid-slicing`) is available for
non-fixed-format documents.

## Troubleshooting

### `ModuleNotFoundError: No module named 'ppocr'`
`tools/train.py` requires the `ppocr` module that ships with
`paddleocr>=2.7.0,<3.0.0`. Make sure you installed `paddleocr` from
`requirements.txt` (not the 3.x inference-only package).

```bash
pip show paddleocr | grep Version   # should be 2.7.x
```

### `PaddleOCR tools/train.py not found`
This means none of the search locations contain the training script.
The local `tools/train.py` bundled with this project is checked first.
If it is missing, reinstall the project from git.

### `RuntimeError: PaddleOCR training failed (return code 1)`
Check the training subprocess output above the error. Common causes:
- `data_dir` or label file path does not exist → re-run `python train.py`
  to regenerate `runs/finetune_v1/data/`
- Not enough disk space for checkpoints

### GPU not detected
```bash
python -c "import paddle; print(paddle.device.cuda.device_count())"
```
If `0`, either CUDA is not installed, the wrong PaddlePaddle variant
was installed, or `--no-gpu` was set. Use `make install-gpu` to reinstall
the GPU variant.

### Windows path errors
Use forward slashes or raw strings in `--images` and `--ground-truths`:
```powershell
python train.py --images "data/scan.gif" --ground-truths "data/gt.xlsx"
```
